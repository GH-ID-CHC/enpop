## 1. 项目概述

### 1.1 目标

在 Windows 任意应用上，通过全局快捷键选中英文文本：

- 调用有道云翻译 API 进行 **英译中** 翻译
- 在鼠标附近弹出 **气泡浮窗** 显示原文与译文
- 支持 **Edge TTS** 自然语音朗读英文原文

### 1.2 用户场景

| 场景 | 说明 |
|------|------|
| 阅读英文文档/文章 | 浏览器中遇到生词或看不懂的句子，选中呼出翻译 |
| IDE 英文报错或注释 | VS Code / IDEA 中快速查看英文提示的含义 |
| PDF 英文文献 | 学术阅读时即选即译，不打断阅读节奏 |
| 英文邮件/消息 | 快速理解不熟悉的英文表达 |

### 1.3 核心原则

- **最小干扰**：浮窗数秒自动消失，用完即走
- **即选即译**：选中文本 → 按快捷键 → 看到翻译，三步完成
- **离线兜底**：翻译需要网络，但 TTS 离线状态下可降级到 Windows 自带语音

### 线程模型

由于 pynput 的全局热键监听会阻塞主线程，而 tkinter 的浮窗也需要事件循环，采用**多线程**设计：

| 线程 | 职责 |
|------|------|
| **主线程** | 系统托盘 (pystray)，常驻等待用户操作 |
| **热键线程** | pynput.Listener 全局键盘监听 |
| **浮窗线程** | 热键触发后启动，运行 tkinter 浮窗 |
| **TTS 线程** | 朗读时不阻塞 UI |

> 浮窗线程需要在热键触发时动态创建，用完后销毁，避免多线程竞争。

#### 热键格式

pynput.GlobalHotKeys 使用字符串表示组合键，例如：

- `"<ctrl>+<alt>+e"` -- Ctrl + Alt + E
- `"<ctrl>+<shift>+t"` -- Ctrl + Shift + T
- `"<ctrl>+<alt>+<shift>+f1"` -- Ctrl + Alt + Shift + F1

#### 关键细节

- 使用 **SendInput API**（替代已弃用的 keybd_event）模拟键盘，兼容性更好
- 只释放 Alt（避免热键 Ctrl+Alt+E 残留导致 Ctrl+Alt+C），**不干预 Ctrl 键**（用户仍按着），保持键状态同步

- 最多重试 3 次，每次重试包含固定延时 + 轮询等待
- 捕获完成后延迟 300ms 恢复原始剪贴板内容

#### API 签名算法

有道使用 appKey + salt + curtime + sign 进行鉴权：

```
sign = sha256(appKey + truncate(text) + salt + curtime + appSecret)
```

其中 truncate(text) 按有道签名规则截断：长度 ≤ 20 用原文，> 20 取 `text[:10] + str(len(text)) + text[-10:]`。

### 5.9 translation_cache.py -- 翻译结果缓存

`python
class TranslationCache:
    """翻译结果本地缓存，持久化到 JSON 文件"""

    CACHE_FILENAME = "translation_cache.json"
    MAX_SIZE = 2000  # 最多缓存条目数

    def __init__(self):
        self._lock = threading.Lock()
        self._cache_path = self._find_cache_path()
        self._cache: dict[str, dict] = {}
        self._load()

    def get(self, text, from_lang="en", to_lang="zh-CHS"):
        """获取缓存结果，未命中返回 None"""

    def set(self, text, from_lang, to_lang, result):
        """存储翻译结果到缓存并持久化"""

    def clear(self):
        """清空所有缓存"""
`

#### 设计说明

TranslationCache 是一个与 ConfigManager 风格一致的轻量本地缓存模块：

- **缓存键**：由 rom_lang:to_lang:normalized_text 构成，normalized_text 为去除首尾空格 + 转小写 + 压缩连续空白后的结果，避免微小排版差异导致缓存未命中
- **持久化**：写入 JSON 文件，与 config.json 同目录（%APPDATA%\EnPop\translation_cache.json），应用重启后缓存仍在
- **线程安全**：使用 	hreading.Lock 保护读写操作，防止并发翻译时的竞争条件
- **容量控制**：MAX_SIZE = 2000，超出时清除较早的一半条目
- **零耦合**：TranslationCache 独立模块，不依赖 YoudaoTranslator 内部实现

#### 工作流

`
翻译请求
  |
  ├── 检查 TranslationCache（缓存命中） ──> 直接显示浮窗
  |
  └── 缓存未命中 ──> 调用 YoudaoTranslator ──> 结果写入缓存 ──> 显示浮窗
`

翻译结果在 main.py 的 _do_translate() 函数中走查缓存路径：

## 7. 有道翻译 API 接入

### 7.1 注册与申请

1. 访问 [有道智云 AI 开放平台](https://ai.youdao.com)
2. 注册账号 -> 控制台 -> 创建应用
3. 选择服务：**文本翻译**
4. 获取 appKey 和 appSecret

### 7.2 API 详情

| 项目 | 值 |
|------|-----|
| 请求地址 | POST https://openapi.youdao.com/api |
| 请求格式 | application/x-www-form-urlencoded |
| 响应格式 | JSON |
| 免费额度 | 注册送 100 元体验金 |
| 计费方式 | 按字符计费（约 48元/百万字符） |

### 7.3 请求参数

| 参数 | 必填 | 说明 |
|------|------|------|
| q | 是 | 待翻译文本（UTF-8 URLEncode） |
| from | 是 | 源语言（auto 自动检测） |
| to | 是 | 目标语言（zh-CHS） |
| appKey | 是 | 应用标识 |
| salt | 是 | 随机字符串 |
| sign | 是 | 签名 sha256(appKey+q+salt+curtime+appSecret) |
| signType | 是 | v3 |
| curtime | 是 | Unix 时间戳（秒） |

### 7.4 响应示例

```json
{
  "errorCode": "0",
  "query": "Hello World",
  "translation": ["你好世界"],
  "basic": {
    "phonetic": "həˈloʊ wɜːrld",
    "explains": ["你好世界"]
  }
}
```

### 7.5 错误码

| 错误码 | 含义 | 处理方式 |
|--------|------|----------|
| 0 | 成功 | -- |
| 101 | 缺少必填参数 | 检查请求参数 |
| 202 | 请求过于频繁或被限制 | 稍后重试 |
| 103 | appKey 无效|| 202 | 请求过于频繁或被限制 | 稍后重试 |
| 103 | appKey 无效 | 检查应用配置 |

### 8.2 配置文件位置（自动管理）

1. 开发模式：%APPDATA%\EnPop\config.json（用户不可见，自动管理）
2. 打包模式：便携优先（exe 同级目录），其次 %APPDATA%\EnPop\config.json
```

配置文件由程序自动读写，用户一般不需要手动编辑。

### 8.3 配置文件格式

```json
{
  "hotkey": "<ctrl>+<alt>+e",
  "tts_engine": "edge",
  "auto_close_seconds": 5,
  "youdao_app_key": "",
  "youdao_app_secret": "",
  "start_on_boot": false
}
```

### 8.4 热键自定义方式

用户通过托盘菜单 -> "设置快捷键..." 打开设置窗口：

1. 弹出一个输入框，提示"按下新的快捷键组合"
2. 程序监听用户按键组合
3. 验证是否与其他全局热键冲突
4. 保存配置并重启热键监听线程

```bash
